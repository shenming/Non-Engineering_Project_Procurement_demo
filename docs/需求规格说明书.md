## 2. 数据架构总览

### 2.1 三层数据模型

```
┌─────────────────────────────────────────────────────────────────┐
│  Layer 1: common/data.js (公共数据层)                           │
│  - METHOD_STEPS        → 步骤名数组（所有采购方式通用）          │
│  - CATEGORY_TREE       → 品类二级树                             │
│  - getStepNames()      → 获取步骤名数组                         │
└──────────────────┬──────────────────────────────────────────────┘
                   │ 被 detail/data.js、edit/index.html 引用
                   ▼
┌─────────────────────────────────────────────────────────────────┐
│  Layer 2: detail/data.js (详情配置+数据层)                      │
│  - DETAIL_METHOD_CONFIG → 步骤状态+节点文件+审批流              │
│    - steps[]            → 带状态的步骤（由 METHOD_STEPS 生成）   │
│    - files[]            → 节点 HTML 文件路径                    │
│    - flow[]             → 审批流数据                            │
│  - DETAIL_DATA          → 完整表单数据                           │
│    - [method].info      → 项目概要信息                          │
│    - [method].nodes[i]  → 每一步的表单+采购+标段+附件数据       │
│  - METHOD_CONFIG        → 别名指向 DETAIL_METHOD_CONFIG         │
└──────────────────┬──────────────────────────────────────────────┘
                   │ 被 edit/data.js、edit/index.html、list.html 引用
                   ▼
┌─────────────────────────────────────────────────────────────────┐
│  Layer 3: edit/data.js (编辑配置层)                             │
│  - EDITABLE_STEPS      → 页面级编辑开关（true=整个页面可编辑）   │
└─────────────────────────────────────────────────────────────────┘
```

## 3. 审批状态与页面可编辑性关系

### 3.1 审批状态定义

`DETAIL_DATA[method].nodes[i].form.approvalStatus` 取值：

| 状态值 | 含义 | 视觉标签 |
|--------|------|----------|
| `审批通过` | 已通过审批，可进入下一步 | `tag-green` |
| `审批中` | 正在OA审批流程中 | `tag-yellow` |
| `草稿` | 尚未提交审批，可编辑 | `tag-gray` |
| `待推进` | 待处理，尚未提交审批 | `tag-gray` |

### 3.2 详情页面权限控制 (detail/index.html)

`updateActionButtons()` 根据 `approvalStatus` 控制按钮：

```
data.nodes[activeStep].form.approvalStatus
         │
         ├── '草稿'  ──→  ✏️编辑启用  │  📤提交启用  │  🗑️删除启用（仅step0）
         ├── '待推进' ──→  ✏️编辑启用  │  📤提交启用  │  🗑️删除禁用
         ├── '审批中' ──→  全部禁用
         └── '审批通过' ──→ 全部禁用
```

### 3.3 编辑页面字段锁定 (edit/index.html)

编辑页面的 `EDIT_LOCKED_FIELDS` 配置（位于 edit/index.html 顶部）定义了每个采购方式每个步骤**哪些字段保持只读**：

```
EDIT_LOCKED_FIELDS = {
    '公开招标': {
        0: [],                             // 所有字段可编辑
        1: [],                             // 所有字段可编辑
        2: [],                             // 所有字段可编辑
        3: [],                             // 所有字段可编辑
        4: [],                             // 所有字段可编辑
        5: [],                             // 所有字段可编辑
        6: []                              // 所有字段可编辑
    },
    '邀请招标': {
        0: ['procCode','companyCode','method','category','approvalStatus','applyDate'],  // 基本信息只读
        1: [],                             // 所有字段可编辑
        2: ['approvalStatus'],             // 除审批状态外可编辑
        3: ['approvalStatus'],             // 除审批状态外可编辑
        4: ['approvalStatus','applyCode','unit']  // 审批状态+定标信息只读
    },
    '询比采购': { /* 同邀请招标 */ },
    '谈判采购': { /* 同邀请招标 */ },
    '直接采购': {
        0: ['procCode','companyCode','method','category','approvalStatus','applyDate'],
        1: ['approvalStatus','applyCode','unit']
    },
    '直接采购-小额': {
        0: ['procCode','companyCode','method','category','approvalStatus','applyDate'],
        1: ['approvalStatus','applyCode','unit']
    }
};
```

**节点页面处理逻辑**（在 `initNode()` 中）：
```javascript
if (params.mode === 'edit') {
    var locked = JSON.parse(sessionStorage.getItem('editLockedFields') || '{}');
    var lockedNames = (locked[method] && locked[method][step]) || [];
    document.querySelectorAll('input,select,textarea').forEach(function(el) {
        var field = el.getAttribute('data-field') || '';
        if (lockedNames.indexOf(field) !== -1) return;  // 保持锁定
        if (el.tagName === 'SELECT') el.disabled = false;
        else el.readOnly = false;
        el.style.background = '';
        el.style.color = '';
        el.style.cursor = '';
    });
}
```

## 5. 采购过程管理

### 5.1 采购过程列表 (list.html)

**功能描述：**
- 以表格形式展示所有采购过程记录
- 支持按状态、采购方式、关键字筛选
- 每行显示：申请编号、项目名称、采购方式、金额、所属单位、当前节点、状态、操作

**当前节点（动态渲染）：**
- 页面加载时 `initTableNodes()` 根据 `data-method` 属性调用 `getActiveStepName(method)` 从 `DETAIL_METHOD_CONFIG` 获取激活的步骤名
- 状态列从 `DETAIL_DATA[method].info.status` 读取
- 草稿状态的记录自动增加删除按钮

### 5.2 创建页面 (create/)

**create/index.html**（一步式）：
- 完整的采购申请表单，含招采信息、采购明细、标段明细、附件管理四大模块
- 通过 sessionStorage `selectedPlanItem` 接收从列表弹框选中的招采项
- 支持直接采购原因选择（D1~D8）
- 提交后跳转回列表

**create/apply.html**（两步式）：
- 步骤1：从 12 条招采项列表中选择一条
- 步骤2：填写申请详情（金额、时间、附件、说明等）
- 支持直接采购情形选择
- 提交后跳转到 `view.html`（旧版详情页，需迁移到 `detail/index.html`）

### 5.3 详情页面 (detail/index.html)

**功能标识：** F-PROC-DETAIL-001

**页面结构：**
- 顶部导航：返回列表 + 页面标题 + 面包屑
- 信息头卡片：项目名称 + 状态标签 + 操作按钮（编辑/删除/提交）
- Tab 页签：招采进度 / 审批流预览
- 招采进度页签：步骤进度条（circle-steps）+ 步骤内容 iframe
- 底部：上一步 / 下一步 / 取消

**步骤渲染逻辑（数据驱动）：**
- `init()` 从 `ctx.method` 读取采购方式 → 设置 `cfg = METHOD_CONFIG[method]`
- `selectedStep` 默认定位到第一个 `state === 'active'` 的步骤
- `renderSteps()` 根据每个步骤的 `state` 和选中态渲染 CSS 类名：

| 条件 | CSS 类 | 视觉 | 可点击 |
|------|--------|------|--------|
| `i === selectedStep` | `.selected` | 🔵 蓝底白字圆点 | ✅ |
| `state === 'done'` 且未选中 | `.done` | 🔵 蓝色边框白底 | ✅ |
| `state === 'active'` 且未选中 | `.done` | 🔵 蓝色边框白底 | ✅ |
| `state === 'future'` | `.future` | ⚪ 灰色 | ❌ |

### 5.4 编辑页面 (edit/index.html)

**功能标识：** F-PROC-EDIT-001

**页面结构：**
- 顶部导航：返回列表 + 页面标题 + 面包屑
- 信息头卡片：项目名称 + 状态标签 + 保存按钮
- Tab 页签：编辑进度 / 审批流预览
- 编辑进度页签：步骤进度条 + 可编辑的步骤内容 iframe
- 底部：上一步 / 下一步 / 返回详情

**与详情页的关键差异：**
| 特性 | detail/index.html | edit/index.html |
|------|-------------------|-----------------|
| 节点路径 | `nodes/xxx.html` | `../detail/nodes/xxx.html` |
| URL 参数 | `?method=&step=` | `?method=&step=&mode=edit` |
| iframe 数据 | `procNodeData` 写入后加载 | `procNodeData` + `editLockedFields` 写入后加载 |
| 保存行为 | 无保存按钮 | 💾 保存 → 回到详情页 |
| 下一步按钮 | ✅ 有 | ✅ 有 |

### 5.5 节点页面权限控制 (detail/nodes/*.html)

每个节点页面在 `mode=edit` 时执行以下逻辑：

```
节点页面加载
    │
    ├─ 从 URL 解析 method, step, mode
    ├─ 从 sessionStorage 读取 procNodeData
    ├─ 渲染表单数据 / 表格 / 附件
    │
    └─ 如果 mode=edit:
        ├─ 读取 sessionStorage.editLockedFields
        ├─ 根据 locked[method][step] 获取锁定字段列表
        ├─ 遍历所有 input/select/textarea:
        │   ├─ 字段在锁定列表 → 保持 readonly/disabled
        │   └─ 字段不在锁定列表 → 解锁（readOnly=false, disabled=false, 恢复样式）
        ├─ 解锁 .op-icon（上传/删除/预览/下载按钮）
        ├─ 添加附件管理工具栏（创建+批量删除）
        └─ 等待 postMessage 数据同步
```

**附件管理工具栏（编辑模式专用）：**
- "＋ 创建"按钮 → 在附件表格末尾添加可编辑新行
- "🗑 批量删除"按钮 → 删除勾选的附件行
- 添加在 `.table-wrap` 父元素之前

## 6. 采购方式配置

### 6.1 步骤状态值

| 状态值 | 含义 | 步骤条样式 | 可点击 |
|--------|------|------------|--------|
| `done` | 已完成 | 蓝框白底 | ✅ |
| `active` | 处理中 | 蓝框白底（未选中）或蓝底白字（选中） | ✅ |
| `future` | 未开始 | 灰色 | ❌ |

### 6.2 各采购方式步骤定义

| 采购方式 | 步骤数 | steps |
|----------|--------|-------|
| 公开招标 | 7 | 招采申请(done)→招采公告(done)→开标信息(done)→上传评标结果(done)→评标结果公示(done)→中标结果(done)→中标结果公示(**active**) |
| 邀请招标 | 5 | 招标申请(done)→拟邀请单位(done)→开标信息(**active**)→上传评标结果(future)→中标结果(future) |
| 询比采购 | 3 | 招标申请(done)→拟邀请单位(**active**)→中标结果(future) |
| 谈判采购 | 3 | 招标申请(done)→拟邀请单位(**active**)→中标结果(future) |
| 直接采购 | 2 | 招标申请(**active**)→确定成交供应商(future) |
| 直接采购-小额 | 2 | 招标申请(done)→确定成交供应商(**active**) |
