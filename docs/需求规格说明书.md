## 1. 总体架构

### 1.1 模块页面结构

```
plans/
├── list.html            # 采购计划列表
├── create.html          # 新建采购计划
├── view.html            # 计划详情（只读）
├── edit.html            # 计划编辑
├── upgrade.html         # 升版
├── version-view.html    # 历史版本详情（只读）
├── item-create.html     # 新增招采项（独立页面）
├── item-edit.html       # 编辑招采项（独立页面）
└── adjust.html          # 调整计划（预留）
```

### 1.2 页面关系图

```
list.html ──「新建」────→ create.html
    │                      └─ 提交审批 → 返回列表
    │
    ├─「查看」───→ view.html（只读详情）
    │               ├──「编辑」──→ edit.html
    │               │               ├── 💾 保存草稿
    │               │               └── ✅ 审批
    │               ├──「作废」→ 发起OA审批
    │               └──「历史版本」→ version-view.html
    │
    └─「升版」───→ upgrade.html
                     ├── 计划信息调整
                     ├── 招采项调整
                     └── 计划说明
```

### 1.3 状态流转

```
[编制中] ──提交审批──→ [审批中] ──OA审批通过──→ [已生效]
                                                  │
                                                  ├── 编辑 → 变更审批
                                                  ├── 升版 → v+1
                                                  └── 作废 → [已作废]

[已生效] ──执行完毕──→ [已完成]
```

## 2. 页面规格

### 2.1 采购计划列表 (list.html)

**功能标识：** F-PLAN-001

**表格列定义：**

| 序号 | 列名 | 字段名 | 数据类型 | 说明 |
|------|------|--------|----------|------|
| 1 | 单选 | — | radio | 用于升版候选选择 |
| 2 | 计划名称 | planName | String | 含编号副文本 |
| 3 | 类型 | planType | Enum | 按年度 / 按事项 |
| 4 | 所属单位 | unit | String | 集团总部 / 成员企业A / B |
| 5 | 计划年度 | year | Integer | 如 2026 |
| 6 | 招采项数 | itemCount | Integer | 该计划下招采项总数 |
| 7 | 版本号 | version | String | v1/v2/... |
 | 8 | 预估总金额 | estimatedAmount | Decimal | 元，如 ¥3,290,000.00元 |
 | 9 | 已执行金额 | executedAmount | Decimal | 元 |
| 10 | 审批状态 | approvalStatus | Enum | 编制中/审批中/已生效 |
| 11 | 执行状态 | execStatus | Enum | 执行中/已完成/已作废/— |
| 12 | 操作 | — | Button | 查看 / 删除 |

**筛选条件：**
- 审批状态：下拉选择（全部 / 编制中 / 审批中 / 已生效）
- 执行状态：下拉选择（全部 / 执行中 / 已完成 / 已作废）
- 计划类型：下拉选择（全部 / 按年度 / 按事项）
- 所属单位：下拉选择
- 计划年度：下拉选择
- 计划名称：文本搜索

**升版联动逻辑：**
- 只允许审批状态=已生效 且 执行状态=执行中或已完成的计划参与升版
- radio 选中后启用升版按钮，否则置灰

**原型演示面板：**
- 角色切换下拉（all / 集团总部 / 成员企业A / 成员企业B）
- 根据 `data-unit` 属性过滤表格行

### 2.2 新建采购计划 (create.html)

**功能标识：** F-PLAN-002

**表单字段：**

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| planName | Text | 是 | — | 计划名称 |
| planType | Select | 是 | 按年度 | 按年度 / 按事项，切换时联动显示年份或事项名称 |
| year | Select | 条件必填 | 2026 | 按年度时必填 |
| eventName | Text | 条件必填 | — | 按事项时必填 |
| unit | Select | 是 | 成员企业A | 集团总部 / 成员企业A / 成员企业B |
 | budget | Number | 是 | 2,000,000.00 | 单位：元 |
| startDate | Date | 是 | 2026-01-01 | — |
| endDate | Date | 是 | 2026-12-31 | — |
| dept | Text | 是 | 采购部 | 编制部门 |
| author | Text | 是 | 张明 | 编制人 |
| description | Textarea | 否 | — | 计划说明 |
| attachment | File | 否 | — | 附件（支持多文件上传） |

**招采项明细表列定义：**

| 列名 | 字段名 | 类型 | 说明 |
|------|--------|------|------|
| 采购内容 | itemName | Text | — |
| 类别 | category | Select | 服务类 / 货物类 |
| 采购方式 | method | Select | 公开招标/邀请招标/询比采购/谈判采购/直接采购 |
 | 预估金额 | estAmount | Number | 单位：元 |
 | 限价 | limitPrice | Number | 单位：元 |
| 计划时间 | schedule | Text | 如 "2026-01 ~ 2026-12" |
| 需求部门 | reqDept | Text | — |
| 经办人 | handler | Text | — |
| 操作 | — | Button | 编辑 / 删除 |

**交互逻辑：**
- 初始空表，显示占位提示
- 点击「＋新增招采项」动态插入一行（全部字段可编辑）
- 保存后以 tag 方式渲染（服务类→蓝色tag，货物类→绿色tag）
- 底部合计实时更新
- 提交时校验至少 1 条招采项

### 2.4 计划编辑 (edit.html)

**功能标识：** F-PLAN-004

- 与 view.html 展示相同信息，但招采项明细表有操作列
- 操作列：已引用行显示 disabled「编辑」「删除」按钮；未引用行可点击编辑/删除

**行内编辑：**

点击「编辑」后，7 个可编辑字段变为 input/select：

| 字段 | 编辑控件 | 说明 |
|------|----------|------|
| 采购内容 | `<input>` text | — |
| 类别 | `<select>` | 服务类/货物类 |
| 采购方式 | `<select>` | 5 种方式 |
 | 采购金额含税（元） | `<input>` number | — |
| 计划时间 | `<input>` text | — |
| 需求部门 | `<input>` text | — |

经办人字段不编辑（保持原值）。

**变更日志：**
- `saveEdit()` 时若值变化，push 到 changeLog[]
- refreshUI() 渲染变更日志表格（变量名称、字段、原值→新值）
- 支持逐条重置 / 全部重置
- 有变更时顶部显示 warning 横幅

**按钮区：** 💾保存草稿 / ✅审批

---

### 2.5 升版 (upgrade.html)

**功能标识：** F-PLAN-005

**三部分卡片：**

**一、计划信息调整：**
- 可编辑字段：计划名称、生效开始/结束日期、总预算、所属单位、计划说明
- 禁用字段：计划类型、计划年度

**二、招采项调整：**
- 表格含完整列头筛选（同 view.html）
- 含导出按钮和新增招采项按钮
- 已引用招采项不可编辑/删除
- 集团管控项不可删除
- 变更日志记录

**三、计划说明：**
- 必填字段：升版原因说明（textarea）
- 附件上传

**按钮区：** 取消（返回列表） / 💾保存草稿 / 🚀提交升版审批

**规则：**
- 版本号 +1
- 提交后旧版本存入历史版本库

### 2.6 历史版本详情 (version-view.html)

**功能标识：** F-PLAN-006

 - 顶部 banner：⚠️ 此版本为历史版本（v1），仅供查看，不可操作
 - 计划概要卡片：状态为「已归档」
 - 招采项明细表无操作列、无筛选控件
 - **调整日志卡片**：展示该版本的调整记录
 - 返回按钮指向 view.html

### 2.7 新增/编辑招采项独立页面

**功能标识：** F-PLAN-007 (item-create.html) / F-PLAN-008 (item-edit.html)

**item-create.html：**
- 表单字段：采购内容、类别（货物类/服务类）、采购方式（5选1）、预估金额、限价、计划开始/结束日期、需求部门、经办人、附件

**item-edit.html：**
- 与 create 同结构，预填原值
- 已上传附件展示

## 4. 通用JS函数

### 4.1 路由导航

```javascript
navTo(module, file)  // 导航到指定模块/文件
// 例: navTo('plans', 'plans/view.html')
// 例: navTo('plans')  // 默认打开模块的 list.html
```

### 4.2 通用弹窗

```javascript
showAlert(message)  // 显示提示弹窗
// 例: showAlert('✅ 草稿已保存，可稍后继续编辑。')
```

### 4.3 列头筛选（view.html / edit.html / upgrade.html 共用）

```javascript
filterAll()  // 收集所有 th-filter 控件值，AND 组合筛选表格行
```

### 4.4 导出（view.html / edit.html / upgrade.html 共用）

```javascript
exportItems()  // 判断是否有筛选条件，提示导出数量
```
