## 2. 数据架构总览

### 2.1 三层数据模型

```
┌─────────────────────────────────────────────────────────────────┐
│  Layer 1: common/data.js (公共数据层)                           │
│  - METHOD_STEPS        → 步骤名数组（所有采购方式通用）          │
│  - CATEGORY_TREE       → 品类二级树                             │
│  - getStepNames()      → 获取步骤名数组                         │
└──────────────────┬──────────────────────────────────────────────┘
                   │ 被 detail/data.js、edit/index.html 引用
                   ▼
┌─────────────────────────────────────────────────────────────────┐
│  Layer 2: detail/data.js (详情配置+数据层)                      │
│  - DETAIL_METHOD_CONFIG → 步骤状态+节点文件+审批流              │
│    - steps[]            → 带状态的步骤（由 METHOD_STEPS 生成）   │
│    - files[]            → 节点 HTML 文件路径                    │
│    - flow[]             → 审批流数据                            │
│  - DETAIL_DATA          → 完整表单数据                           │
│    - [method].info      → 项目概要信息                          │
│    - [method].nodes[i]  → 每一步的表单+采购+标段+附件数据       │
│    - [method].supplements → 补充招标记录（公开招标/邀请招标）    │
│  - METHOD_CONFIG        → 别名指向 DETAIL_METHOD_CONFIG         │
└──────────────────┬──────────────────────────────────────────────┘
                   │ 被 edit/data.js、edit/index.html、list.html、supplementary/list.html 引用
                   ▼
┌─────────────────────────────────────────────────────────────────┐
│  Layer 3: edit/data.js (编辑配置层)                             │
│  - EDITABLE_STEPS      → 页面级编辑开关（true=整个页面可编辑）   │
└─────────────────────────────────────────────────────────────────┘
```

## 3. 审批状态与页面可编辑性关系

### 3.1 审批状态定义

`DETAIL_DATA[method].nodes[i].form.approvalStatus` 取值：

| 状态值 | 含义 | 视觉标签 |
|--------|------|----------|
| `审批通过` | 已通过审批，可进入下一步 | `tag-green` |
| `审批中` | 正在OA审批流程中 | `tag-yellow` |
| `草稿` | 尚未提交审批，可编辑 | `tag-gray` |
| `待推进` | 待处理，尚未提交审批 | `tag-gray` |

`DETAIL_DATA[method].info.status` 额外取值（采购记录整体状态）：

| 状态值 | 含义 | 视觉标签 |
|--------|------|----------|
| `进行中` | 采购流程进行中 | `tag-yellow` |
| `已完成` | 采购已完成 | `tag-green` |
| `草稿` | 尚未提交 | `tag-gray` |
| `弃标` | 已弃标（通过弃标流标功能设置） | `tag-gray` |
| `流标` | 已流标（通过弃标流标功能设置） | `tag-gray` |

### 3.2 详情页面权限控制 (detail/index.html)

`updateActionButtons()` 根据 `approvalStatus` 控制按钮：

```
data.nodes[activeStep].form.approvalStatus
         │
         ├── '草稿'  ──→  ✏️编辑启用  │  📤提交启用  │  🗑️删除启用（仅step0）
         ├── '待推进' ──→  ✏️编辑启用  │  📤提交启用  │  🗑️删除禁用
         ├── '审批中' ──→  全部禁用
         └── '审批通过' ──→ 全部禁用
```

### 3.3 编辑页面字段锁定 (edit/index.html)

编辑页面的 `EDIT_LOCKED_FIELDS` 配置（位于 `edit/data.js`）定义了每个采购方式每个步骤**哪些字段保持只读**。

**节点页面处理逻辑**（在 `initNode()` 中）：
```javascript
if (params.mode === 'edit') {
    var locked = JSON.parse(sessionStorage.getItem('editLockedFields') || '{}');
    var lockedNames = (locked[method] && locked[method][step]) || [];
    document.querySelectorAll('input,select,textarea').forEach(function(el) {
        var field = el.getAttribute('data-field') || '';
        if (lockedNames.indexOf(field) !== -1) return;  // 保持锁定
        if (el.tagName === 'SELECT') el.disabled = false;
        else el.readOnly = false;
        el.style.background = '';
        el.style.color = '';
        el.style.cursor = '';
    });
}
```

### 3.4 EDITABLE_STEPS 编辑开关 (edit/data.js)

## 5. 采购过程管理

### 5.1 采购过程列表 (list.html)

**功能描述：**
- 以表格形式展示所有采购过程记录
- 支持按状态、采购方式、关键字筛选（状态筛选项含「弃标」「流标」）
- 每行显示：申请编号、项目名称、采购方式、金额、所属单位、当前节点、状态、操作
- 顶部操作：📥 导出、＋ 发起申请、🏳 弃标流标

**当前节点（动态渲染）：**
- 页面加载时 `initTableNodes()` 根据 `data-method` 属性调用 `getActiveStepName(method)` 从 `DETAIL_METHOD_CONFIG` 获取激活的步骤名
- 状态列从 `DETAIL_DATA[method].info.status` 读取
- 草稿状态的记录自动增加删除按钮

**发起申请弹窗：**
- 点击「＋ 发起申请」弹出 `#planSelectModal`
- 展示 `PLAN_ITEMS` 数组中的招采项（仅已生效且未被引用）
- 选中招采项后 → `sessionStorage.selectedPlanItem` → `navTo('procurement', 'procurement/create.html')`

### 5.2 创建页面 (create/index.html)

**功能描述：**
- 完整的采购申请表单，含招采信息、采购明细、标段明细、附件管理四大模块
- 通过 sessionStorage `selectedPlanItem` 接收从列表弹框选中的招采项
- 支持直接采购原因选择（D1~D8）
- 提交后跳转回列表

### 5.3 详情页面 (detail/index.html)

**页面结构：**
- 顶部导航：返回列表 + 页面标题 + 面包屑
- 信息头卡片：项目名称 + 状态标签 + 操作按钮（编辑/删除/提交）
- Tab 页签：招采进度 / 审批流预览
- 招采进度页签：步骤进度条（circle-steps）+ 步骤内容 iframe
- 底部：上一步 / 下一步 / 取消

**步骤渲染规则：**

| 条件 | CSS 类 | 视觉 | 可点击 |
|------|--------|------|--------|
| `i === selectedStep` | `.selected` | 🔵 蓝底白字圆点 | ✅ |
| `state === 'done'` 且未选中 | `.done` | 🔵 蓝色边框白底 | ✅ |
| `state === 'active'` 且未选中 | `.done` | 🔵 蓝色边框白底 | ✅ |
| `state === 'future'` | `.future` | ⚪ 灰色 | ❌ |

### 5.4 编辑页面 (edit/index.html)

**页面结构：**
- 顶部导航：返回详情 + 页面标题 + 面包屑
- 信息头卡片：项目名称 + 状态标签 + 保存按钮
- Tab 页签：编辑进度 / 审批流预览
- 编辑进度页签：步骤进度条 + 可编辑的步骤内容 iframe
- 底部：上一步 / 下一步 / 返回详情

### 5.5 节点页面 (detail/nodes/*.html)

7 个节点页面按采购方式的步骤映射动态加载，公共逻辑：

- 从 URL 解析 `?method=&step=&mode`
- `sessionStorage.getItem('procNodeData')` → 取对应 method/step 的数据
- 渲染表单 + 采购明细表格 + 标段明细表格 + 附件表格
- `mode=edit` 时解锁非锁定字段 + 显示附件管理工具栏 + 支持采购明细/标段编辑

### 5.6 弃标流标功能 (bid-abandon.html)【v3.0 新增】

**功能标识：** F-PROC-ABANDON-001

**入口：** 采购过程列表 → 🏳弃标流标

**流程：**

```
列表页 → 点击🏳弃标流标
  → #abandonModal 弹窗显示可选采购申请
    → 筛选规则:
      ├─ 遍历 DETAIL_DATA 所有节点
      ├─ 排除任一节点 form.approvalStatus === '审批中' 的记录
      └─ 排除 info.status 已为 '弃标'/'流标' 的记录
  → 选中后 → abandonProcData 写入 sessionStorage
  → navTo('procurement', 'procurement/bid-abandon.html')
      ├─ 卡1：招采信息（14个字段只读展示）
      ├─ 卡2：采购明细（只读表格）
      ├─ 卡3：标段明细（只读表格）
      ├─ 卡4：附件管理（只读表格）
      └─ 卡5：弃标流标原因
          ├─ 类型选择：🏳弃标 / 🌊流标（二选一，必选）
          ├─ 原因 textarea（必填）
          └─ 附件上传（可选）
  → 提交审核:
      ├─ 校验类型和原因必填
      ├─ 二次确认弹窗
      ├─ 更新 DETAIL_DATA[methodKey].info.status = '弃标'/'流标'
      ├─ 更新 planItemRefStatus[planItemId] = '未引用'
      └─ 返回列表页
```

**`canAbandon()` 过滤逻辑：**

```javascript
function canAbandon(methodKey) {
    var data = DETAIL_DATA[methodKey];
    if (!data || !data.nodes) return false;
    // 已弃标/流标的不再显示
    if (data.info && (data.info.status === '弃标' || data.info.status === '流标')) return false;
    for (var key in data.nodes) {
        var node = data.nodes[key];
        if (node.form && node.form.approvalStatus === '审批中') return false;
    }
    return true;
}
```
