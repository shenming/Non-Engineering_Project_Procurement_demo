## 2. 采购过程管理

### 2.1 采购过程列表 (list.html)

**功能描述：**
- 以表格形式展示所有采购过程记录
- 支持按状态、采购方式、关键字筛选
- 每行显示：申请编号、项目名称、采购方式、金额、所属单位、当前节点、状态、操作

**当前节点（动态渲染）：**
- 页面加载时根据 `data-method` 属性调用 `getActiveStepName(method)` 自动从 `METHOD_CONFIG` 获取激活的步骤名
- `METHOD_CONFIG` 位于 `common/data.js`，步骤定义如下：

```javascript
METHOD_CONFIG = {
    '公开招标': {
        steps: [
            { name: '招采申请',     state: 'done' },
            { name: '招采公告',     state: 'done' },
            { name: '开标信息',     state: 'done' },
            { name: '上传评标结果', state: 'done' },
            { name: '评标结果公示', state: 'done' },
            { name: '中标结果',     state: 'done' },
            { name: '中标结果公示', state: 'active' }
        ],
        files: [...],
        flow: [...]
    },
    '邀请招标': {
        steps: [
            { name: '招标申请', state: 'done' },
            { name: '发出邀请', state: 'done' },
            { name: '开标',     state: 'active' },
            { name: '评标',     state: 'future' },
            { name: '中标结果', state: 'future' }
        ],
        files: [...],
        flow: [...]
    }
}
```

### 2.2 详情主页面 (detail/index.html)

**功能标识：** F-PROC-DETAIL-001

**页面结构：**
- 顶部导航：返回列表 + 页面标题 + 面包屑
- 信息头卡片：项目名称 + 状态标签 + 操作按钮（编辑/删除/提交）
- Tab 页签：招采进度 / 审批流预览
- 招采进度页签：步骤进度条（circle-steps）+ 步骤内容 iframe
- 底部：上一步 / 下一步 / 取消

**步骤渲染逻辑（数据驱动）：**
- `init()` 从 `ctx.method` 读取采购方式 → 设置 `cfg = METHOD_CONFIG[method]`
- `doneCount` 和 `currentStep` 从 `cfg.steps[].state` 读取
- `renderSteps()` 根据每个步骤的 `state` 渲染 CSS 类名：
  - `done` → `.done`（蓝框白底）
  - `active` → `.active`（蓝底白字）
  - `future` → `.future`（灰色）

**审批流预览（数据驱动）：**
- `renderApprovalFlow()` 从 `cfg.flow` 动态渲染表格
- 每行包含：节点描述、审批状态（tag标签）、审批日期、流程 ID

**节点加载：**
- `loadNode(index)` 将 `DETAIL_DATA` 存入 `sessionStorage`
- iframe 通过 URL 参数 `?method=&step=` 获取当前采购方式和步骤
- 节点页面从 `sessionStorage` 读取数据进行动态渲染

### 2.3 节点页面 (detail/nodes/*.html)

**数据来源：** 所有 7 个节点页面统一从 `sessionStorage` 读取 `procNodeData`

```javascript
var params = getUrlParams();
var method = params.method || '公开招标';
var step = parseInt(params.step) || 0;
var pd = JSON.parse(sessionStorage.getItem('procNodeData') || '{}');
var nodeData = pd[method].nodes[step];
```

**各节点数据字段：**

| 节点 | 页面 | form 字段 | 附加数据 |

| 1 | bid-announce.html | announceName, startDate, endDate, approvalStatus | attachments[] |
| 2 | bid-opening.html | desc, bidderCount, approvalStatus | attachments[] |
| 3 | eval-result.html | desc, approvalStatus | attachments[] |
| 4 | eval-announce.html | desc, approvalStatus | attachments[] |
| 5 | award-result.html | approvalStatus, applyCode, unit, resultNote | sections[], attachments{} |

## 3. 数据层架构

### 3.1 公共数据层 (`common/data.js`)

| 变量/函数 | 用途 |
|-----------|------|
| `CATEGORY_TREE` | 采购类别二级品类树 |
| `METHOD_CONFIG` | 采购方式配置（步骤+文件+审批流） |
| `getStepNames(method)` | 获取采购方式的步骤名数组 |
| `getActiveStepName(method)` | 获取当前激活的步骤名 |
| `getDoneCount(method)` | 获取已完成步骤数 |

### 3.2 详情数据层 (`detail/data.js`)

| 变量 | 用途 |
|------|------|
| `DETAIL_DATA['公开招标']` | 公开招标详情数据（7个节点） |
| `DETAIL_DATA['邀请招标']` | 邀请招标详情数据（5个节点） |
| `DETAIL_DATA[method].info` | 顶部信息（projectName, applyCode, unit, status） |
| `DETAIL_DATA[method].nodes[N]` | 第 N 个节点的完整数据 |

### 3.3 数据流

```
list.html → sessionStorage(procurementViewContext)
  → detail/index.html init() 读取 ctx.method
    → 设置 cfg = METHOD_CONFIG[method]
      → loadNode() 存 DETAIL_DATA 到 sessionStorage
        → iframe 加载节点 → 从 sessionStorage 读取数据
          → 动态渲染表单 + 表格 + 附件
```
