## 1. 引言

### 1.1 编写目的
本文档旨在描述非工程类招采管理系统的用户需求，明确各模块功能范围、业务流程与操作规范，作为后续需求规格说明与开发实现的基础。

### 1.2 适用范围
本文档适用于非工程类招采管理系统的原型设计与开发团队，覆盖仪表盘、采购计划、采购过程、报表管理、配置管理五大模块。

### 1.3 系统导航

系统采用左侧栏导航 + iframe 内容区的主框架布局（`index.html`），当前显示 6 个菜单项：

| 菜单项 | 模块 | 说明 |
|--------|------|------|
| 📊 仪表盘 | `dashboard` | 统计概览、三色预警、采购动态 |
| 📋 采购计划 | `plans` | 计划编制、审批、升版、作废 |
| ⚡ 采购过程 | `procurement` | 招采申请、详情/编辑、弃标流标 |
| 📈 报表管理 | `reports` | 统计报表、自定义报表 |
| ⚙️ 配置管理 | `settings` | 阈值、字典、预警规则 |

## 2. 仪表盘

### 2.1 仪表盘首页 (dashboard/index.html)

**入口：** 系统默认首页

**功能描述：**

1. **统计卡片**（4 张）：
   - 📋 采购计划：当前计划总数
   - 📦 招采项：全部招采项数量
   - ⚡ 进行中采购：正在执行的采购数量
   - 🔴 逾期预警：需立即处理的红灯项数量

2. **📅 年度计划概览**：
   - 表格展示所有年度计划的基本信息（名称、类型、招采项数、金额、完成率、状态）
   - 完成率以进度条可视化展示
   - → 查看全部 跳转采购计划列表

3. **🚨 预警看板 · 三色预警**：
   - 汇总标签：绿灯/黄灯/红灯数量统计
   - 表格展示预警项：所属计划、招采项、单位、计划时间、状态
   - 红灯项引导至计划详情页处理

4. **📌 近期采购动态**：
   - 展示最近 3 条采购记录（关联计划、编号、项目名、方式、金额、状态）
   - → 查看全部 跳转采购过程列表

## 3. 采购计划管理

### 3.1 采购计划列表

**入口：** 采购计划管理 → 计划列表

**功能描述：**
- 以表格形式展示所有最新版本采购计划
- 列表永远只显示各计划的最新版本
- 支持按审批状态、执行状态、计划类型、所属单位、年度筛选
- 支持按计划名称搜索
- 显示字段：计划名称（含编号）、类型、所属单位、计划年度、招采项数、版本号、预估总金额（限价合计）、审批状态、执行状态、操作

**操作按钮：**
| 按钮 | 说明 | 可见条件 |
|------|------|----------|
| 查看 | 进入计划详情页 | 始终可见 |
| 删除 | 删除编制中的计划 | 审批状态=编制中时可见 |
| 升版 | 对选中计划发起升版 | 审批=已生效 且 执行=执行中或已完成时可选 |

**列表页面顶部操作：**
- 📥 导出当前列表
- ＋ 新建计划
- 🚀 升版（需先勾选符合条件的计划）

**原型演示面板（可折叠）：** 切换查看视角（集团用户 / 集团总部 / 成员企业A / 成员企业B），按所属单位过滤数据。

### 3.2 新建采购计划

**入口：** 计划列表 → ＋新建计划

**功能描述：**
- 填写计划基本信息：名称、类型（按年度/按事项）、所属单位、预算、生效日期、编制部门/人、计划说明
- 上传附件（预算批复、需求说明等佐证材料）
- 添加招采项明细（通过内联表格逐条录入）
- 招采项字段：采购内容、类别（货物类/服务类）、采购方式（公开招标/邀请招标/询比采购/谈判采购/直接采购）、采购项说明（谈判/直接采购时必填）、采购金额含税（元）、计划时间、需求部门、经办人
- 可逐条编辑、删除招采项
- 底部实时显示限价合计
- 提交时校验：至少 1 条招采项，且限价合计不得超过计划总预算

**采购方式金额门槛校验（`validateItemMethod`）：**
- 谈判采购、直接采购：不受金额门槛限制，需选择采购项说明情形
- 询比采购：货物类＜100万元，服务类＜50万元
- 邀请招标：＜公开招标下限（货物类200万元/服务类100万元）
- 公开招标：不限金额门槛
- 如所选采购方式不满足金额门槛，保存时拦截并提示原因

**采购项说明校验（`validateItemNote`）：**
- 谈判采购：必须选择一种情形（技术复杂/性质特殊、多种实施方案、仅2家合格供应商）
- 直接采购：必须选择一种情形（7种情形之一），若选D1（合同估算价＜100,000.00元）则金额必须＜100,000元

**操作按钮：**
- 💾 保存草稿
- 🚀 提交审批
- 取消（返回列表）

### 3.3 计划详情（查看）

**入口：** 计划列表 → 查看

**功能描述：**
- 页面顶部显示计划名称、版本号、操作按钮（作废/编辑）
- 采用 Tab 页签布局，分为 3 个页签：
  - **📋 基本信息**：核心字段（编号、类型、年度、单位、日期、预算、部门/人、审批状态、执行状态）及计划说明
  - **📜 版本管理**：当前版本信息、调整日志列表、历史版本表格（含调整日志摘要）
  - **📎 附件与审批**：审批状态、执行状态、OA审批单号、附件文件列表

**列头筛选：** 支持按类别、采购方式、采购项说明、采购金额区间、计划时间区间、需求部门、经办人、引用状态、状态进行组合筛选

**操作按钮（卡片顶部）：**
| 按钮 | 说明 |
|------|------|
| 🗑️ 作废 | 发起OA审批作废计划（校验无执行中的招采项） |
| ✏️ 编辑 | 进入编辑模式 |

**汇总统计：** 招采项总数、限价合计、计划总预算剩余（超出时红色高亮）

### 3.4 计划编辑

**入口：** 计划详情 → ✏️编辑

**功能描述：**
- 展示计划概要信息（只读）
- 招采项明细表格支持行内编辑、删除、新增
- 已引用（已发起招标）的招采项禁止编辑和删除
- 未引用的招采项可直接在表格行内编辑
- 编辑后记录变更日志，支持逐条或全部重置
- 变更日志展示变更字段、原值、新值
- 变更项通过审批后生效

**行内校验：** 保存编辑时同样执行 `validateItemMethod` 和 `validateItemNote` 校验

### 3.5 升版

**入口：** 计划列表 → 勾选计划 → 🚀升版

**前置条件：** 审批状态=已生效，执行状态=执行中或已完成

**功能描述：**
升版页面分三部分：

**一、计划信息调整：**
- 可修改计划名称、生效日期、总预算、所属单位、计划说明
- 禁用字段：计划类型、计划年度

**二、招采项调整：**
- 招采项明细表格支持列头筛选
- 含导出按钮和新增招采项按钮
- 已引用招采项禁止编辑，未引用招采项可编辑和删除
- 变更日志记录所有修改

**三、计划说明（升版原因）：**
- 必填，说明本次升版调整的原因、内容和依据
- 可上传调整依据附件

**规则说明：**
- 版本号 +1（如 v2→v3）
- 提交后走OA审批流程，审批通过后自动替换当前生效版本

### 3.6 历史版本查看

**入口：** 计划详情 → 历史版本 → 查看详情

**功能描述：**
- 以只读形式展示历史版本的全部信息
- 顶部提示 "此版本为历史版本（v1），仅供查看，不可操作"
- 展示该历史版本的调整日志

### 3.7 作废计划

**入口：** 计划详情 → 🗑️作废

**前置条件：** 无执行中的招采项

**功能描述：**
- 触发OA审批流程
- 审批通过后计划状态变为「已作废」，不可恢复

## 4. 采购过程管理

### 4.1 采购过程列表 (procurement/list.html)

**入口：** 采购过程 → 列表页

**功能描述：**
- 以表格形式展示所有采购过程记录（10 条 demo 数据）
- 支持按状态、采购方式、关键字筛选
- 状态筛选项：全部、草稿、审批中、进行中、已完成、弃标、流标
- 每行显示：申请编号、项目名称、采购方式、金额、所属单位、当前节点、状态、操作
- 当前节点从 `DETAIL_METHOD_CONFIG` 动态获取激活步骤名
- 状态从 `DETAIL_DATA[method].info.status` 读取

**列表顶部操作：**

| 按钮 | 功能 |
|------|------|
| 📥 导出 | 导出当前列表 |
| ＋ 发起申请 | 弹出招采项选择弹窗 → 跳转创建页 |
| 🏳 弃标流标 | 弹出采购申请选择弹窗 → 跳转弃标流标填报页 |

**发起申请弹窗（`#planSelectModal`）：**
- 展示 `PLAN_ITEMS` 中的招采项（仅已生效且未被引用）
- 列：选择、计划编号、采购内容、所属单位、采购类别、预选采购方式、预估金额、采购限价、经办人
- 支持计划编号和采购内容模糊搜索
- 选中后 → `sessionStorage.selectedPlanItem` → 跳转创建页

**行操作：**
- 查看：跳转详情页（携带 method、projectName 等上下文）
- 删除（仅草稿状态可见）：删除记录并恢复关联计划项为「未引用」

### 4.2 创建采购申请 (procurement/create/index.html)

**入口：** 列表页 → ＋发起申请 → 选择招采项

**功能描述：**
- 一步式完整采购申请表单
- 从 `sessionStorage.selectedPlanItem` 获取选中的招采项信息
- 四大模块：招采信息表单、采购明细表格、标段明细表格、附件管理
- 支持直接采购情形选择（D1~D8）
- 提交后返回列表页

### 4.3 详情页面 (procurement/detail/index.html)

**入口：** 列表页 → 查看

**功能描述：**
- 顶部导航：返回列表 + 面包屑（项目名称）
- 信息头：项目名称 + 状态标签 + 操作按钮
- Tab 页签：📌 招采进度 / ✅ 审批流预览
- 招采进度：步骤进度条（circle-steps）+ 步骤内容 iframe
- 步骤进度条支持点击切换（已完成的步骤可回看）
- 底部：上一步 / 下一步 / 取消

**操作按钮权限：**
| 当前步骤状态 | ✏️编辑 | 📤提交 | 🗑️删除 |
|-------------|--------|--------|--------|
| 草稿 | ✅ | ✅ | ✅（仅step0） |
| 待推进 | ✅ | ✅ | ❌ |
| 审批中 | ❌ | ❌ | ❌ |
| 审批通过 | ❌ | ❌ | ❌ |

**步骤进度条：**
- 已完成/处理中步骤：蓝色圆点，可点击
- 当前选中步骤：蓝底白字
- 未来步骤：灰色，不可点击

### 4.4 编辑页面 (procurement/edit/index.html)

**入口：** 详情页 → ✏️编辑

**功能描述：**
- 与详情页相同的步骤条 + iframe 布局
- 步骤内容以 `mode=edit` 加载节点页面
- 节点页面根据 `EDIT_LOCKED_FIELDS` 配置解锁可编辑字段
- 采购明细和标段明细支持编辑（金额、标段数可变，自动校验标段汇总与金额一致）
- 附件管理显示工具栏（创建 + 批量删除）
- 💾 保存 → 校验标段数据 → 返回详情页

### 4.5 弃标流标 (procurement/bid-abandon.html)【v2.0 新增】

**入口：** 列表页 → 🏳弃标流标 → 选择采购申请

**前置条件：** 所选采购申请无「审批中」节点，且非弃标/流标状态

**选择弹窗（`#abandonModal`）：**
- 列出所有可弃标的采购申请（自动过滤审批中和已弃标/流标）
- 列：选择、申请编号、项目名称、采购方式、金额、所属单位、当前节点、状态
- 支持关键字搜索

**填报页面（`bid-abandon.html`）：**
- 卡1 招采信息：14 个字段只读展示（编号、名称、公司、限额、方式、品类、经办人、日期、代理机构、是否进场等）
- 卡2 采购明细：只读表格
- 卡3 标段明细：只读表格
- 卡4 附件管理：只读表格
- 卡5 弃标流标原因：
  - 🏳弃标 / 🌊流标 类型选择（二选一，必选）
  - 原因 textarea（必填）
  - 附件上传（可选）

**提交效果：**
- 招采申请状态置为「弃标」或「流标」
- 关联采购计划项恢复为「未引用」，可重新发起招采申请
- 已弃标/流标的记录在弃标流标弹窗中不再显示
- 操作不可撤销

### 4.6 补充招标 (supplementary/list.html)【功能保留，侧边栏隐藏】

**功能描述：**
- 仅展示公开招标和邀请招标的补充记录
- 数据源：`DETAIL_DATA[method].supplements[]`
- 每条记录显示：项目名称、补充次数、状态、编号、日期、原因、变更内容对比、附件
- → 查看 跳转 `supplementary/detail.html`

## 7. 状态说明

### 7.1 计划审批状态
| 状态 | 含义 |
|------|------|
| 编制中 | 计划正在编制，尚未提交审批 |
| 审批中 | 计划已提交，正在OA审批流转中 |
| 已生效 | 审批通过，计划正式生效 |

### 7.2 计划执行状态
| 状态 | 含义 |
|------|------|
| 执行中 | 有招采项正在招标或执行中 |
| 已完成 | 所有招采项均已完成招标 |
| 已作废 | 计划已被作废，不可恢复 |

### 7.3 采购记录状态
| 状态 | 含义 |
|------|------|
| 草稿 | 尚未提交审批 |
| 进行中 | 采购流程进行中 |
| 已完成 | 采购已完成 |
| 弃标 | 已弃标（通过弃标流标功能设置） |
| 流标 | 已流标（通过弃标流标功能设置） |

### 7.4 节点审批状态
| 状态 | 含义 |
|------|------|
| 审批通过 | 已通过审批 |
| 审批中 | 正在OA审批流程中 |
| 草稿 | 尚未提交审批 |
| 待推进 | 待处理 |

### 7.5 招采项引用状态
| 状态 | 含义 |
|------|------|
| 已引用 | 已发起招标流程，不可编辑或删除 |

## 9. 金额校验规则

### 9.1 采购方式门槛校验 (`validateItemMethod`)

| 采购方式 | 货物类 | 服务类 | 特殊说明 |
|----------|--------|--------|----------|
| 公开招标 | 不限金额 | 不限金额 | 企业可自主选择 |
| 邀请招标 | ＜200万元 | ＜100万元 | 上限=公开招标下限 |
| 询比采购 | ＜100万元 | ＜50万元 | — |
| 谈判采购 | 不限金额 | 不限金额 | 需选择适用情形 |
| 直接采购 | 不限金额 | 不限金额 | 需选择适用情形，D1情形金额＜10万 |

### 9.2 采购项说明校验 (`validateItemNote`)

- **谈判采购**：必须选择（技术复杂/性质特殊、多种实施方案、仅2家合格供应商）之一
- **直接采购**：必须选择7种情形之一，选D1时金额必须＜100,000元

### 9.3 计划预算校验

- 新建计划提交时：校验各招采项限价合计是否超出计划总预算
- 超出时拦截提交，提示具体超出金额
