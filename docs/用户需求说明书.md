## 1. 引言

### 1.1 编写目的
本文档旨在描述非工程类招采管理系统中「采购计划」模块的用户需求，明确功能范围、业务流程与操作规范，作为后续需求规格说明与开发实现的基础。

### 1.2 适用范围
本文档适用于采购计划模块的原型设计与开发团队，覆盖集团总部及下属成员企业的采购计划编制、审批、执行、升版、作废等全生命周期管理。

---

## 2. 业务概述

采购计划模块用于管理和跟踪集团及下属成员企业的年度或专项采购计划。核心业务流程包括：

1. 编制采购计划，添加招采项明细
2. 提交审批，经OA流程生效
3. 已生效计划可进行编辑调整、升版操作
4. 计划执行中/完成后可查询历史版本
5. 计划可作废（通过OA审批）

## 3. 功能需求

### 3.1 采购计划列表

**入口：** 采购计划管理 → 计划列表

**功能描述：**
- 以表格形式展示所有最新版本采购计划
- 列表永远只显示各计划的最新版本
- 支持按审批状态、执行状态、计划类型、所属单位、年度筛选
- 支持按计划名称搜索
- 显示字段：计划名称（含编号）、类型、所属单位、计划年度、招采项数、版本号、预估总金额、已执行金额、审批状态、执行状态、操作

**操作按钮：**
| 按钮 | 说明 | 可见条件 |
|------|------|----------|
| 查看 | 进入计划详情页 | 始终可见 |
| 删除 | 删除编制中的计划 | 审批状态=编制中时可见 |
| 升版 | 对选中计划发起升版 | 审批=已生效 且 执行=执行中或已完成时可选 |

**列表页面顶部操作：**
- 📥 导出当前列表
- ＋ 新建计划
- 🚀 升版（需先勾选符合条件的计划）

**原型演示面板（可折叠）：** 切换查看视角（集团用户 / 集团总部 / 成员企业A / 成员企业B），按所属单位过滤数据。

### 3.2 新建采购计划

**入口：** 计划列表 → ＋新建计划

**功能描述：**
- 填写计划基本信息：名称、类型（按年度/按事项）、所属单位、预算、生效日期、编制部门/人、计划说明
- 上传附件（预算批复、需求说明等佐证材料）
- 添加招采项明细（通过内联表格逐条录入）
- 招采项字段：采购内容、类别（货物类/服务类）、采购方式（公开招标/邀请招标/询比采购/谈判采购/直接采购）、采购项说明（谈判/直接采购时必填）、采购金额含税（元）、计划时间、需求部门、经办人
- 可逐条编辑、删除招采项
- 底部实时显示限价合计
- 提交时校验：至少 1 条招采项，且限价合计不得超过计划总预算

**采购方式金额门槛校验（`validateItemMethod`）：**
- 谈判采购、直接采购：不受金额门槛限制，需选择采购项说明情形
- 询比采购：货物类＜100万元，服务类＜50万元
- 邀请招标：＜公开招标下限（货物类200万元/服务类100万元）
- 公开招标：不限金额门槛
- 如所选采购方式不满足金额门槛，保存时拦截并提示原因

**采购项说明校验（`validateItemNote`）：**
- 谈判采购：必须选择一种情形（技术复杂/性质特殊、多种实施方案、仅2家合格供应商）
- 直接采购：必须选择一种情形（7种情形之一），若选D1（合同估算价＜100,000.00元）则金额必须＜100,000元

**操作按钮：**
- 💾 保存草稿
- 🚀 提交审批
- 取消（返回列表）

**规则说明：**
- 新建计划生成 v1 版本
- 提交后计划状态变为「审批中」
- 审批通过后进入「已生效」状态
- 已保存招采项可点击编辑进入编辑模式，修改后重新保存

### 3.3 计划详情（查看）

**入口：** 计划列表 → 查看

**功能描述：**
- 页面顶部显示计划名称、版本号、操作按钮（作废/编辑）
- 采用 Tab 页签布局，分为 3 个页签：
  - **📋 基本信息**：核心字段（编号、类型、年度、单位、日期、预算、部门/人、审批状态、执行状态）及计划说明
  - **📜 版本管理**：当前版本信息、调整日志列表、历史版本表格（含调整日志摘要）
  - **📎 附件与审批**：审批状态、执行状态、OA审批单号、附件文件列表

**列头筛选：** 支持按类别、采购方式、采购项说明、采购金额区间、计划时间区间、需求部门、经办人、引用状态、状态进行组合筛选

**操作按钮（卡片顶部）：**
| 按钮 | 说明 |
|------|------|
| 🗑️ 作废 | 发起OA审批作废计划（校验无执行中的招采项） |
| ✏️ 编辑 | 进入编辑模式 |

**汇总统计：** 招采项总数、限价合计、计划总预算剩余（超出时红色高亮）

---

### 3.4 计划编辑

**入口：** 计划详情 → ✏️编辑

**功能描述：**
- 展示计划概要信息（只读）
- 招采项明细表格支持行内编辑、删除、新增
- 已引用（已发起招标）的招采项禁止编辑和删除
- 未引用的招采项可直接在表格行内编辑
- 编辑后记录变更日志，支持逐条或全部重置
- 变更日志展示变更字段、原值、新值
- 变更项通过审批后生效

**行内校验：** 保存编辑时同样执行 `validateItemMethod`（类别+金额→采购方式门槛）和 `validateItemNote`（采购项说明合规性）

**操作按钮：**
- 💾 保存草稿
- ✅ 审批（提交变更审批，变更通过后即时生效）

---

### 3.5 升版

**入口：** 计划列表 → 勾选计划 → 🚀升版

**前置条件：** 审批状态=已生效，执行状态=执行中或已完成

**功能描述：**
升版页面分三部分：

**一、计划信息调整：**
- 可修改计划名称、生效日期、总预算、所属单位、计划说明
- 禁用字段：计划类型、计划年度

**二、招采项调整：**
- 招采项明细表格支持列头筛选（与详情页一致）
- 含导出按钮和新增招采项按钮
- 已引用招采项禁止编辑，集团管控项禁止删除
- 未引用招采项可编辑和删除
- 变更日志记录所有修改
- 行内编辑/新增时同样执行金额门槛校验和采购项说明校验

**三、计划说明（升版原因）：**
- 必填，说明本次升版调整的原因、内容和依据
- 可上传调整依据附件

**操作按钮：**
- 取消（返回列表）
- 💾 保存草稿
- 🚀 提交升版审批

**规则说明：**
- 版本号 +1（如 v2→v3）
- 提交后走OA审批流程
- 审批通过后自动替换当前生效版本
- 旧版本存入历史版本库

---

### 3.6 历史版本查看

**入口：** 计划详情 → 历史版本 → 查看详情

**功能描述：**
- 以只读形式展示历史版本的全部信息
- 顶部提示 "此版本为历史版本（v1），仅供查看，不可操作"
- 招采项明细无操作列
- 展示该历史版本的调整日志
- 返回按钮指向当前生效版本详情页

---

### 3.7 作废计划

**入口：** 计划详情 → 🗑️作废

**前置条件：** 无执行中的招采项

**功能描述：**
- 触发OA审批流程
- 审批通过后计划状态变为「已作废」，不可恢复

## 4. 状态说明

### 4.1 审批状态
| 状态 | 含义 |
|------|------|
| 编制中 | 计划正在编制，尚未提交审批 |
| 审批中 | 计划已提交，正在OA审批流转中 |
| 已生效 | 审批通过，计划正式生效 |

### 4.2 执行状态
| 状态 | 含义 |
|------|------|
| 执行中 | 有招采项正在招标或执行中 |
| 已完成 | 所有招采项均已完成招标 |
| 已作废 | 计划已被作废，不可恢复 |

### 4.3 招采项状态
| 状态 | 含义 |
|------|------|
| 已生效 | 招采项已生效，尚未开始招标 |
| 执行中 | 正在招标中 |
| 已完成 | 招标已完成 |
| 草稿 | 尚未提交审批 |

### 4.4 引用状态
| 状态 | 含义 |
|------|------|
| 已引用 | 已发起招标流程，不可编辑或删除 |
| 未引用 | 尚未发起招标，可编辑和删除 |

### 4.5 版本状态
| 状态 | 含义 |
|------|------|
| 已生效 | 当前活跃版本 |
| 已归档 | 历史版本，仅供查看 |

## 6. 权限说明（原型演示）

- 集团用户可查看所有单位的采购计划
- 下属企业用户仅可查看本单位的采购计划
- 编制人具备编辑草稿和提交审批权限
- 非编制人仅可查看

## 7. 金额校验规则

### 7.1 采购方式门槛校验 (`validateItemMethod`)

| 采购方式 | 货物类 | 服务类 | 特殊说明 |
|----------|--------|--------|----------|
| 公开招标 | 不限金额 | 不限金额 | 企业可自主选择 |
| 邀请招标 | ＜200万元 | ＜100万元 | 上限=公开招标下限 |
| 询比采购 | ＜100万元 | ＜50万元 | — |
| 谈判采购 | 不限金额 | 不限金额 | 需选择适用情形 |
| 直接采购 | 不限金额 | 不限金额 | 需选择适用情形，D1情形金额＜10万 |

### 7.2 采购项说明校验 (`validateItemNote`)

- **谈判采购**：必须选择（技术复杂/性质特殊、多种实施方案、仅2家合格供应商）之一
- **直接采购**：必须选择7种情形之一，选D1（合同估算价＜100,000.00元）时金额必须＜100,000元

### 7.3 计划预算校验

- 新建计划提交时：校验各招采项限价合计是否超出计划总预算
- 超出时拦截提交，提示具体超出金额
- `submitCreate()` 遍历所有已保存招采项限价，与预算输入值比较
